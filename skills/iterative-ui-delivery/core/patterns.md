# 模式

## 模式 1：生成与演进分离

- 首版页面用“快速生成模式”达成可看可点
- 后续修改用“演进模式”逐轮打磨
- 重点从“首版是否完美”转为“每轮是否稳态变好”

## 模式 2：反馈结构化

把“感觉不对”“看着别扭”改写成：
- 改哪里（区域/组件）
- 改成什么（可见结果）
- 如何验收（对比标准）

## 模式 3：门禁前置

不要求一开始完备需求，但每轮必须过 Gate：
- 构建可通过
- 风格不漂移
- 交互不回归
- 目录不失控

## 模式 4：共性组件先行

当同类控件出现 2 次及以上（如下拉、按钮组、筛选栏）：
- 抽为共享组件
- 禁止继续复制页面私有版本
