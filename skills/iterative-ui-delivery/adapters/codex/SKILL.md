# 迭代式前端交付

当用户需求不完整、经常边看边改，且希望每轮迭代都有明确质量门禁时，使用这个 skill。

## 默认输出顺序

1. 本轮 Mini Brief（目标/用户动作/验收信号）
2. 最小改动实现方案
3. 改动后验证结果（build/ux/behavior/boundary）
4. 本轮变更记录
5. 下一轮建议
