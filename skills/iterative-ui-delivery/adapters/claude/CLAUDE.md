# 迭代式前端交付

面向需求不完整、边做边改场景的前端迭代交付与每轮质量门禁工作流。

## 意图

# 意图

## 一句话说明

在需求不完整或持续变更的情况下，用“小步快跑 + 每轮质量门禁”稳定交付前端页面。

## 适用场景

- 创业团队或快速试错团队，需求经常边做边变
- 产品表达偏口语化，需要先出可看版本再收敛
- 前端工作量主要来自“后续修改完善”而非首版搭建

## 默认假设

- 无法一次性拿到完整 PRD
- 每轮迭代都能获得业务反馈（哪怕只有一句）
- 团队接受“先可用、后完善”的增量交付方式

## 典型产出

- 每轮 1 个可演示的小增量页面能力
- 每轮一份变更记录（目标/交付/反馈/下一步）
- 持续可运行、可构建、风格一致的页面基线

## 工作流

# 工作流

## 标准流程

1. **Mini Brief（3 行）**
- 本轮目标：只写一个可观察改进
- 用户动作：本轮用户能完成什么操作
- 验收信号：看到什么现象算完成

2. **小步实现**
- 只改一个垂直切片（页面区域/交互块）
- 非阻塞问题不做大重构

3. **快速评审**
- 立即展示结果
- 把口语反馈改写为“可执行改动项”

4. **增量修正**
- 仅修改受影响模块
- 未被点名的行为默认保持不变

5. **每轮 Gate（强制）**
- Build Gate：`npm run build`
- UX Gate：样式一致性（间距、颜色、组件、字体、状态）
- Behavior Gate：变更交互端到端可用
- Boundary Gate：目录边界清晰（app/pages/features/shared）

## 关键判断点

- 反馈是否可执行：不可执行就先改写再开发
- 变更是否过大：过大就继续拆轮次
- 是否引入风格漂移：有漂移先对齐再继续功能

## 输出约定

1. 先给本轮目标与完成状态
2. 再列具体改动文件与行为变化
3. 最后给下一轮建议（1~3条）

## 模式

# 模式

## 模式 1：生成与演进分离

- 首版页面用“快速生成模式”达成可看可点
- 后续修改用“演进模式”逐轮打磨
- 重点从“首版是否完美”转为“每轮是否稳态变好”

## 模式 2：反馈结构化

把“感觉不对”“看着别扭”改写成：
- 改哪里（区域/组件）
- 改成什么（可见结果）
- 如何验收（对比标准）

## 模式 3：门禁前置

不要求一开始完备需求，但每轮必须过 Gate：
- 构建可通过
- 风格不漂移
- 交互不回归
- 目录不失控

## 模式 4：共性组件先行

当同类控件出现 2 次及以上（如下拉、按钮组、筛选栏）：
- 抽为共享组件
- 禁止继续复制页面私有版本

## 风格

# 风格

## 风格原则

- 统一优先于炫技：先保证整站一致，再做局部亮点
- 延续已有视觉语言：颜色、圆角、边框、阴影、状态统一
- 禁止“系统默认控件穿帮”：原生控件与页面风格不一致时，改为自定义组件

## 控件一致性要求

- 按钮：主次按钮层级明确，hover/focus/disabled 状态完整
- 下拉：触发器与弹层统一风格，不使用系统默认弹出样式
- 表单：输入框、下拉、文本域边框与焦点态保持一套 token
- 表格：表头、行 hover、操作列风格一致

## 风格变更策略

- 每轮最多引入一类新视觉模式
- 新模式先在一个页面试点，通过后再推广

## 校验

# 校验

## 最低检查项（每轮必做）

- Build Gate 通过（`npm run build`）
- 变更功能可手动走通
- 页面无明显风格割裂
- 未引入目录层级失控或跨层依赖

## 校验场景

- 理想场景：反馈明确，单轮可闭环
- 模糊场景：反馈抽象，需先结构化再开发
- 边界场景：需求变更影响多个模块，需拆轮并保持可回退

## 通过条件

- 本轮目标已达成且可演示
- 本轮没有构建错误与明显回归
- 已产出下一轮待办（可直接接着做）

## 安装说明

# 安装说明

## 仓库内维护

- 主源只维护 `skill.yaml` 与 `core/*.md`
- adapter 文件由 `./scripts/build <skill-id>` 生成
- 不直接手改 adapters，避免与 core 漂移

## 推荐流程

1. 修改 `core/*.md`
2. 执行 `./scripts/build iterative-ui-delivery`
3. 执行 `./scripts/doctor iterative-ui-delivery`
4. 再执行 `./scripts/install <tool> iterative-ui-delivery`

## 注意

- `codex` 安装会链接到 `~/.codex/skills/<skill-id>`
- `claude`、`cursor` 走各自 adapter 安装路径
