# 视觉输入

## 目的

这个文件用于帮助“不懂设计术语、但知道自己想要什么感觉”的用户，把模糊视觉诉求翻译成明确的前端约束。

## 用户常见输入方式

用户可能通过这些方式描述风格：

- 直接说形容词
- 给参考产品
- 贴截图或 UI 图片
- 用否定句表达不要什么，例如 `不要花哨`

## 翻译规则

不要要求用户先学设计黑话。应该把日常表达翻译成具体决策。

例如：

- `稳重`
  - 低饱和配色
  - 浅中性色背景
  - 点缀色使用克制
  - 间距节奏规则
- `专业`
  - 层级清晰
  - 表格和表单结构可读
  - 装饰元素少
- `高级一点`
  - 颜色更少
  - 间距更统一
  - 阴影更轻
  - 字体层级更克制
- `像后台`
  - 筛选区明确
  - 以表格和表单为主
  - 导航清楚
  - 交互更紧凑
- `不要大屏感`
  - 避免深色驾驶舱
  - 避免夸张巨图表
  - 避免发光渐变
  - 避免装饰性过强
- `不要太丑`
  - 降低随机感
  - 统一圆角和间距
  - 避免混杂视觉语言
- `蓝色忧郁`
  - 低饱和蓝灰主色
  - 冷静、略克制的情绪
  - 适中的信息密度，而不是空旷豪华感
  - 高亮克制，不要霓虹蓝
- `不要太土`
  - 避免廉价渐变
  - 避免杂乱多色
  - 降低花式图表修饰
  - 统一排版和间距

## 参考产品处理规则

用户提到知名产品时，不要模仿品牌，只提炼可复用特征。

例如：

- `飞书后台`
  - 冷静的企业协作感
  - 清爽间距
  - 中等信息密度
  - 实用的卡片与表格
- `阿里云控制台`
  - 强平台感
  - 更密的导航和管理结构
  - 状态展示清楚
  - 操作导向更强

## 输出格式

风格相关时，优先产出这张卡片：

```md
视觉决策卡
- 风格关键词：
- 参考气质：
- 配色方向：
- 字体语气：
- 信息密度：
- 导航方式：
- 表格风格：
- 表单风格：
- 图表风格：
- 明确不要：
```

## 非设计背景用户规则

如果用户只说 `你帮我定`，不要停下来。直接从下面的预设里选一个最稳的企业中后台方案，并明确告知你的选择。

# 风格预设

## 预设 A：稳重企业型

适用表达：

- 稳重
- 专业
- 企业中后台
- 不要花哨

特征：

- 浅中性色背景
- 蓝色或石板灰点缀
- 中等信息密度
- 卡片克制
- 筛选区明确
- 表格可读性优先
- 弱阴影、中等圆角

避免：

- 巨型 hero
- 深色驾驶舱
- 多彩渐变

## 预设 B：平台控制台型

适用表达：

- 平台型
- 控制台
- 像云平台
- 运维后台

特征：

- 更强的导航层级
- 更紧凑的布局
- 状态优先
- 卡片更紧
- 表格占比更高
- 辅助面板更明确

避免：

- 官网式叙事块
- 过大的留白

## 预设 C：轻量运营型

适用表达：

- 轻量
- 简洁
- 日常运营
- 不要太重

特征：

- 更多呼吸感
- 边框层级更少
- KPI 卡数量更少
- 图表处理更克制

避免：

- 过重的控制台密度
- 过多堆叠卡片

## 预设 D：审核工作台型

适用表达：

- 审核
- 审批
- 可追溯
- 风控

特征：

- 队列 + 详情分栏
- 明确的动作区
- 强调时间线与审计信息
- 状态和风险标识清楚
- 整体冷静但严肃

避免：

- 把关键操作藏进普通下拉菜单
- 用装饰性仪表盘代替工作面

## 预设 E：配置工作台型

适用表达：

- 配置
- 映射
- 模板
- 版本

特征：

- 分组表单或左右工作台
- 吸底操作栏
- 校验状态清晰可见
- 版本与审计信息显式展示
- 通过分组控制表单密度

避免：

- 巨型无分组表单
- 把校验和发布动作埋得太深

## 预设 F：蓝灰忧郁型后台

适用表达：

- 蓝色忧郁
- 偏蓝灰
- 不要太土
- 信息密度适中

特征：

- 低饱和蓝灰主色
- 中浅背景 + 冷静表面层
- 中等偏上的信息密度
- 表格和图表都克制，不追求秀肌肉
- 状态色依然清晰，但主色不艳

避免：

- 霓虹蓝
- 科技大屏感
- 花哨渐变块
- 炫技仪表盘布局
